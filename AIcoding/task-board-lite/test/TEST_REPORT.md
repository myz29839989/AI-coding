## 3. 手工冒烟建议

1. 启动后端：`cd backend && npm start`。
2. 用浏览器打开 `frontend/index.html`。
3. 创建两个团队 `QA-A`、`QA-B`。
4. 进入 `QA-A` 创建任务 `A-only-task`。
5. 切换到 `QA-B`，确认看不到 `A-only-task`。
6. 将任务从待办拖到进行中，刷新后确认状态保留。
7. 删除团队，确认主控面板不再展示该团队。

## 4. 剩余风险

- JSON 文件存储只适合本地 MVP，不适合多实例部署。
- 当前并发保护是单进程写入队列 + `version` 乐观锁；如果部署多个 Node 进程，需要迁移到数据库或集中式锁。
- 前端使用原生 HTML5 拖拽，移动端体验有限。

## 5. 测试结论

当前版本满足“简洁轻便、最短时间实现”的 MVP 验收要求。核心风险点团队隔离、状态更新、旧版本冲突和删除级联已由后端自动化测试覆盖。
